# Examples

Each subdirectory is a self-contained example that demonstrates a feature or
pattern from `@baetheus/css`.

## Creating a new example

1. Create a new directory under `examples/` named after your example (e.g.
   `examples/my-feature`).
2. Add a `style.ts` that imports from the library, builds your styles, and
   exports the rendered CSS string as a named export called `css`:

   ```ts
   import { render, style } from "../mod.ts";

   const heading = style("h1", { fontSize: "2rem" });

   export const css = render(heading);
   ```

3. Add an `index.html` that links to `style.css` (generated by the build
   script) and demonstrates the styles in the browser.

The directory must contain **both** `style.ts` and `index.html` to be picked
up by the build script and the dev server index.

## `build.ts` — compile styles to CSS

```
deno run --allow-read --allow-write --allow-env build.ts
```

Scans every subdirectory of `examples/` for directories that contain both a
`style.ts` and an `index.html`. For each match it dynamically imports
`style.ts` and, if the module exports a string named `css`, writes that string
to `style.css` in the same directory. Directories without a `css` string
export are skipped with a warning.

Run this whenever you change a `style.ts` to regenerate the corresponding
`style.css` before committing or serving.

## `main.ts` — dev server

```
deno run --allow-read --allow-net main.ts
```

Starts an HTTP file server rooted at the `examples/` directory. On startup it
scans the same directories as the build script and generates an in-memory
`index.html` at `/` that links to every discovered example. All other requests
are served directly from the filesystem, so the compiled `style.css` files (and
any other static assets) are available without an extra build step as long as
you have run `build.ts` at least once.
