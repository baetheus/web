Vertical Rhythm

A worked example showing how @baetheus/css establishes a consistent baseline grid for body text and headings.

What is vertical rhythm?

Vertical rhythm is the practice of aligning every vertical measurement on a page — line heights, margins, padding — to a single base unit derived from body text. The result is text that visually sits on a regular grid no matter how many headings, paragraphs, or asides appear between sections.

In this example the base unit is 1.5rem, calculated as the body font size of 1rem multiplied by a line height of 1.5. Every margin, line height, and heading size below is a whole multiple of that unit. Try resizing the window or adding paragraphs in the devtools and the rhythm holds.

Step one: define a theme

The base unit and font scale are declared once as a theme. A theme is a plain object passed to theme(); each leaf becomes a CSS custom property, and the returned value is a typed tree of var() references. 

import { theme } from "@baetheus/css";

const vr = theme({
  font: {
    family: "Verdana, Geneva, sans-serif",
    size: "1rem",
  },
  leading: "1.5rem",
  scale: {
    xs:   "0.75rem",
    sm:   "0.875rem",
    md:   "1rem",
    lg:   "1.25rem",
    xl:   "1.5rem",
    xxl:  "2rem",
    xxxl: "2.5rem",
  },
});

Reading vr.leading later returns a var(−−hash−leading) reference. The actual values are emitted by passing the theme to style(":root", vr), which produces the :root block at the top of the generated stylesheet.

Step two: style semantic elements

With the theme in hand, each block-level element gets a style whose line height and margins are multiples of vr.leading. The body sets the baseline; headings increase their font size while keeping their line height on the grid; paragraphs reset top margin and add one unit of bottom margin. 

import { style } from "@baetheus/css";

const body = style("body", {
  fontFamily: vr.font.family,
  fontSize:   vr.font.size,
  lineHeight: vr.leading,
  margin:     0,
  padding:    `${vr.leading} calc(${vr.leading} * 2)`,
});

const h1 = style("h1", {
  fontFamily:   vr.font.family,
  fontSize:     vr.scale.xxxl,
  lineHeight:   "3rem",         // 2 base units
  fontWeight:   "bold",
  marginTop:    "3rem",         // 2 base units
  marginBottom: vr.leading,     // 1 base unit
  padding:      0,
});

Every heading on this page — including the <h1> at the top — uses a style built exactly like the one above. Because the selectors target the bare elements, no class attributes are needed in the markup.

Step three: render the stylesheet

Individual styles are combined with join() and turned into a CSS string by render(). Writing that string to a .css file produces the stylesheet linked in this page’s <head>. 

import { join, render } from "@baetheus/css";

export const css = render(
  join(root, body, h1, h2, h3, p, small, /* utilities... */),
);

The render() output is deterministic and content hashed, so class names stay stable across runs and the CSS can be committed alongside the source.

The rhythm in action

Headings stay on the grid

This <h3> above uses the vr.scale.xl font size (1.5rem) and the base line height. Because both values are multiples of the base unit, the heading occupies exactly one grid row.

Long paragraphs keep their rhythm

Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur.

Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum. Each line of body text advances by exactly one base unit, so the bottoms of paragraphs in adjacent columns — if you placed them side by side — would align.

Small text still aligns

The <small> element drops the font size to vr.scale.sm but keeps the base line height, so even fine print sits on the same grid as body copy. That trick is what keeps captions, footnotes, and legal text from disrupting the rhythm.

Utility classes

For one-off adjustments the example exports a handful of margin and text-size utilities. Each is created by calling style() with no selector, which auto-generates a content-hashed class name. 

const mt0 = style({ marginTop: 0 });
const mt1 = style({ marginTop: vr.leading });
const mt2 = style({ marginTop: `calc(${vr.leading} * 2)` });

const textSm = style({ fontSize: vr.scale.sm, lineHeight: vr.leading });
const textLg = style({ fontSize: vr.scale.lg, lineHeight: vr.leading });

To apply one, read its generated class name with mt2.toString() or pass several to use() to build a space-separated string suitable for the class attribute.
